# Purpose

You are a Higher Order Prompt (HOP) that creates implementation plans with QA integrations expertise as the primary focus. It reads the integrations expertise file and critical related files before delegating to `/plan`, ensuring the planning process is informed by accurate knowledge of each integration's data shapes, env vars, caller contracts, and gotchas.

## Variables

USER_REQUEST: $1
PRIOR_SPEC: $2
EXPERTISE_FILE: .claude/commands/mental-model/experience-integrations/expertise.yaml

## Instructions

- This is a two-step wrapper: load expertise context, then delegate to `/plan`
- The expertise file is your **mental model** — treat it as authoritative but verify against the codebase
- If PRIOR_SPEC is set, read it first as the upstream contract before loading the expertise file
- Let `/plan` handle the actual planning logic while you provide domain context
- Prioritize integration-specific elements: env var names, data shape contracts, caller isolation rules

## Workflow

1. **Load Upstream Spec (conditional)**
   - If PRIOR_SPEC is set and non-empty: read that file first as the upstream contract
   - Note any integration-touching requirements or constraints from the prior spec

2. **Load Expertise Context**
   - Read EXPERTISE_FILE to understand integration architecture, data shapes, env vars, and gotchas
   - Identify which integrations (WCS, AOS, OSI, Adobe I/O, webhooks, Jira, Slack) are relevant to USER_REQUEST
   - Read the relevant source files to confirm current implementation state:
     - For price validation: wcs.js, aos.js, osi-mapping.js
     - For webhook/AEM: webhooks/index.js, adobe-io.js
     - For notifications: jira.js, slack.js
   - Note which server service is the caller for each integration (runner.js, webhooks action)
   - Surface relevant gotchas — especially: OSI vs offerId confusion, env var naming mismatches, RUNNER_MODE suppression

3. **Execute Planning with Context**
   - Call `/plan` with USER_REQUEST as the argument
   - Planning will now be informed by the QA integrations expertise loaded above

## Report

The report will be generated by the `/plan` command.
